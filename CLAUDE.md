# PartsSource Design System — Claude Instructions

## Role
You are a design system engineer for PartsSource. Your job is to generate production-ready React components, Storybook stories, and tests from design tokens. Every output must be pixel-accurate to the Figma design and accessible.

---

## Tech Stack
- **Framework**: React 18 + TypeScript
- **Styling**: Tailwind CSS (token-driven via `tailwind-tokens.js`)
- **Component library**: Custom (no MUI, no Chakra)
- **Storybook**: v7+
- **Testing**: Jest + React Testing Library + Axe + Playwright
- **Token source**: `src/tokens/tokens.ts` (auto-generated by Style Dictionary)

---

## Token Rules — CRITICAL

### Always use tokens. Never use raw values.

❌ WRONG:
```tsx
<button className="bg-[#005ba7] text-white px-10 py-4 rounded text-base font-semibold">
```

✅ RIGHT:
```tsx
<button className="bg-[var(--ps-semantic-color-brand-primary)] text-[var(--ps-semantic-color-surface-default)] px-[var(--ps-button-primary-paddingH)] py-[var(--ps-button-primary-paddingV)] rounded-[var(--ps-button-primary-radius)] text-base font-semibold">
```

Or using the TypeScript token map:
```tsx
import { tokens } from '@/tokens/tokens';
// tokens['semantic.color.brand.primary'] === '#005ba7'
```

### Token hierarchy to follow:
1. **Component tokens first**: `button.primary.default.background`
2. **Semantic tokens second**: `semantic.color.brand.primary`
3. **Primitive tokens never directly in components**: only via semantic references

### Token naming pattern:
`{level}.{category}.{variant}.{state}.{property}`

---

## Component Requirements

Every component you generate MUST include:

### 1. All states
- `default` (resting)
- `hover`
- `pressed` / `active`
- `focused` (visible focus ring — WCAG 2.4.7)
- `disabled` (non-interactive, opacity applied)
- `loading` (if applicable)
- `error` (if applicable)

### 2. All variants from the token file
Read the component's JSON token file to identify variants. The top-level keys in the token JSON are the variant groups.

### 3. Props interface
Build the props interface from the token file structure. Use the actual variant names found in the token file — do not invent variants or omit ones that exist.

## Button Family — one token file, multiple component files

`tokens/ps-tokens/component/button.json` contains several distinct button families. Generate a **separate React component file** for each family listed below. They all live under `src/components/Button/`:

| Token group in `button.json` | React component file | Variant prop values |
|---|---|---|
| `button.primary` `button.secondary` `button.tertiary` `button.ghost` `button.danger` | `Button.tsx` | `'primary' \| 'secondary' \| 'tertiary' \| 'ghost' \| 'danger'` |
| `button.inline.director` `button.inline.underline` `button.inline.allCaps` | `InlineButton.tsx` | `'director' \| 'underline' \| 'allCaps'` |

| `button.square` | `IconButton.tsx` | `'lg' \| 'sm'` — mark with `/** @deprecated LEGACY orange CTA — do not use in new work */` |

### 4. Accessibility
- Semantic HTML elements (`<button>`, not `<div>`)
- `aria-disabled` (not just `disabled` attribute alone)
- `aria-busy` for loading states
- Visible focus ring using `focus-visible:` Tailwind utilities
- Color contrast meets WCAG AA (4.5:1 for text, 3:1 for UI components)
- Minimum touch target: 44×44px (`semantic.sizing.touch.min`)

## Component Template

```tsx
// src/components/Button/Button.tsx
import React from 'react';
import { tokens } from '@/tokens/tokens';

export interface ButtonProps {
  variant?: 'primary' | 'secondary' | 'tertiary' | 'ghost' | 'danger';
  size?: 'sm' | 'lg';
  disabled?: boolean;
  loading?: boolean;
  leftIcon?: React.ReactNode;
  rightIcon?: React.ReactNode;
  onClick?: (e: React.MouseEvent<HTMLButtonElement>) => void;
  children: React.ReactNode;
  className?: string;
  'aria-label'?: string;
}

export const Button: React.FC<ButtonProps> = ({
  variant = 'primary',
  size = 'lg',
  disabled = false,
  loading = false,
  leftIcon,
  rightIcon,
  onClick,
  children,
  className = '',
  'aria-label': ariaLabel,
}) => {
  const isDisabled = disabled || loading;

  const baseClasses = [
    'inline-flex items-center justify-center gap-2',
    'font-semibold uppercase tracking-[0.5px]',
    'transition-colors duration-150',
    'focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-offset-2',
    'focus-visible:ring-[var(--ps-semantic-color-brand-primary)]',
    'rounded-[var(--ps-button-primary-radius)]',
  ].join(' ');

  const variantClasses: Record<string, string> = {
    primary: [
      'bg-[var(--ps-button-primary-default-background)]',
      'text-[var(--ps-button-primary-default-text)]',
      'border-2 border-transparent',
      'hover:bg-[var(--ps-button-primary-hover-background)]',
      'active:bg-[var(--ps-button-primary-pressed-background)]',
      'disabled:bg-[var(--ps-button-primary-disabled-background)]',
      'disabled:text-[var(--ps-button-primary-disabled-text)]',
    ].join(' '),
    secondary: [
      'bg-[var(--ps-button-secondary-default-background)]',
      'text-[var(--ps-button-secondary-default-text)]',
      'border-2 border-[var(--ps-button-secondary-default-border)]',
      'hover:bg-[var(--ps-button-secondary-hover-background)]',
      'hover:text-[var(--ps-button-secondary-hover-text)]',
      'hover:border-[var(--ps-button-secondary-hover-border)]',
      'active:bg-[var(--ps-button-secondary-pressed-background)]',
      'active:border-[var(--ps-button-secondary-pressed-border)]',
      'disabled:opacity-40',
    ].join(' '),
    tertiary: [
      'bg-[var(--ps-button-tertiary-default-background)]',
      'text-[var(--ps-button-tertiary-default-text)]',
      'border-0',
      'rounded-full',
      'hover:bg-[var(--ps-button-tertiary-hover-background)]',
      'disabled:opacity-40',
    ].join(' '),
    ghost: [
      'bg-transparent',
      'text-[var(--ps-button-ghost-default-text)]',
      'border-0',
      'hover:bg-[var(--ps-button-ghost-hover-background)]',
      'active:bg-[var(--ps-button-ghost-pressed-background)]',
      'disabled:opacity-40',
    ].join(' '),
    danger: [
      'bg-[var(--ps-button-danger-default-background)]',
      'text-[var(--ps-button-danger-default-text)]',
      'border-2 border-transparent',
      'hover:bg-[var(--ps-button-danger-hover-background)]',
      'active:bg-[var(--ps-button-danger-pressed-background)]',
      'disabled:opacity-40',
    ].join(' '),
  };

  const sizeClasses: Record<string, string> = {
    lg: 'h-[var(--ps-button-primary-height-lg)] px-[var(--ps-button-primary-paddingH-lg)] text-base',
    sm: 'h-[var(--ps-button-primary-height-sm)] px-[var(--ps-button-primary-paddingH-sm)] text-sm',
  };

  return (
    <button
      type="button"
      className={[
        baseClasses,
        variantClasses[variant],
        sizeClasses[size],
        isDisabled ? 'opacity-40 cursor-not-allowed pointer-events-none' : '',
        className,
      ].join(' ')}
      disabled={isDisabled}
      aria-disabled={isDisabled}
      aria-busy={loading}
      aria-label={ariaLabel}
      onClick={!isDisabled ? onClick : undefined}
    >
      {loading ? (
        <>
          <span className="sr-only">Loading</span>
          <svg className="animate-spin h-4 w-4" viewBox="0 0 24 24" fill="none" aria-hidden="true">
            <circle className="opacity-25" cx="12" cy="12" r="10" stroke="currentColor" strokeWidth="4" />
            <path className="opacity-75" fill="currentColor" d="M4 12a8 8 0 018-8v8H4z" />
          </svg>
        </>
      ) : (
        <>
          {leftIcon && <span className="inline-flex" aria-hidden="true">{leftIcon}</span>}
          {children}
          {rightIcon && <span className="inline-flex" aria-hidden="true">{rightIcon}</span>}
        </>
      )}
    </button>
  );
};

export default Button;
```

## Test Template

```tsx
// src/components/Button/Button.test.tsx
import { render, screen, fireEvent } from '@testing-library/react';
import { Button } from './Button';

describe('Button', () => {
  it('renders children', () => {
    render(<Button>Add to Cart</Button>);
    expect(screen.getByRole('button', { name: /add to cart/i })).toBeInTheDocument();
  });

  it('calls onClick when clicked', () => {
    const onClick = jest.fn();
    render(<Button onClick={onClick}>Click me</Button>);
    fireEvent.click(screen.getByRole('button'));
    expect(onClick).toHaveBeenCalledTimes(1);
  });

  it('does not call onClick when disabled', () => {
    const onClick = jest.fn();
    render(<Button onClick={onClick} disabled>Disabled</Button>);
    fireEvent.click(screen.getByRole('button'));
    expect(onClick).not.toHaveBeenCalled();
  });

  it('shows loading state correctly', () => {
    render(<Button loading>Submit</Button>);
    expect(screen.getByRole('button')).toHaveAttribute('aria-busy', 'true');
    expect(screen.getByText('Loading')).toBeInTheDocument();
  });

  it('applies aria-disabled when disabled', () => {
    render(<Button disabled>Disabled</Button>);
    expect(screen.getByRole('button')).toHaveAttribute('aria-disabled', 'true');
  });
});
```

## Rules Summary

| Rule | Detail |
|------|--------|
| No raw values | Use CSS vars or token imports |
| All states | default, hover, pressed, focused, disabled, loading |
| All variants | Read from component token JSON |
| Semantic HTML | `<button>`, `<nav>`, `<input>` — not `<div>` |
| WCAG AA | Color contrast, focus visible, touch targets |
| Storybook | One story per state + AllVariants story |
| Tests | Unit (Jest) + Accessibility (Axe) per component |
| TypeScript | Strict types, no `any` |
| File structure | Component / Stories / Tests / index.ts |

---

## What NOT to do

- ❌ No `style={{ color: '#005ba7' }}` — use CSS variables
- ❌ No hardcoded colors in Tailwind — `bg-blue-600` is WRONG
- ❌ No missing states — every variant needs all interactive states
- ❌ No skipping accessibility — axe test is mandatory
- ❌ No `div` pretending to be a button — use semantic HTML
- ❌ No raw pixel values — use spacing/sizing tokens

---

## Token file location

When generating a component, read:
- `tokens/ps-tokens/component/{componentName}.json` — component-specific tokens
- `src/tokens/tokens.css` — CSS custom properties (reference for var names)
- `src/tokens/tokens.ts` — TypeScript token map (reference for JS usage)
